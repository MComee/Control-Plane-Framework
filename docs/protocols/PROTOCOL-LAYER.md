# Protocol Layer — Control Plane Framework

## Overview

Control Plane Framework includes a formal protocol layer based on Genesis Omni 2.7 ST.

This layer defines:

- reasoning constraints
- decomposition rules
- validation requirements
- execution discipline

## Location

Protocol artifacts are located in:

`docs/protocols/genesis-omni/`

## Structure

- Canonical protocol (full source)
- Framework adoption mapping
- Protocol status and inheritance rules

## Role in the system

The protocol layer:

- governs how work is interpreted
- constrains how plans are formed
- defines validation expectations
- establishes consistency across tools and workflows

The framework itself:

- organizes these rules into repository structure

Projects built on the framework:

- apply these rules in a specific implementation context

## Critical distinction

Protocol defines behavior.
Framework defines structure.
Projects define implementation.

All three must remain aligned.
