# Sample Project Brief

## Project name
Example Project

## One-line description
A sample application used to demonstrate how a new project can inherit the framework and begin with an established control structure.

## Problem statement
The project needs a clean startup pattern that supports planning, governance, execution, and validation before major implementation begins.

## Goals
- demonstrate use of the planning lane
- show how governance constraints are documented
- map project work into execution artifacts
- define milestone acceptance criteria

## Non-goals
- production-ready implementation
- stack-specific code generation
- vendor-specific workflow lock-in

## Initial stack assumptions
- repository hosting via Git
- one or more chat-based AI tools
- one or more command-line or local AI tools
- human review before milestone acceptance

## Success criteria
- the project mission is clear
- constraints are visible
- active tasks are documented
- acceptance criteria exist before milestone completion
